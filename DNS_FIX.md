# DNS 解析问题修复方案

## 问题原因
自定义 DNS 服务器 `30.30.30.30` 无法访问，导致解析失败。

## 推荐的公共 DNS 服务器

### 国内 DNS（速度快）
- **114.114.114.114** - 114DNS（推荐）
- **223.5.5.5** - 阿里云 DNS
- **119.29.29.29** - 腾讯 DNS
- **180.76.76.76** - 百度 DNS

### 国外 DNS（稳定性高）
- **8.8.8.8** - Google DNS（推荐）
- **1.1.1.1** - Cloudflare DNS
- **208.67.222.222** - OpenDNS

## 测试命令

```bash
# 测试 Google DNS
dig @8.8.8.8 center.qoder.sh

# 测试 114DNS
dig @114.114.114.114 center.qoder.sh

# 测试阿里云 DNS
dig @223.5.5.5 center.qoder.sh
```

## 修复步骤

1. 将自定义 DNS 从 `30.30.30.30` 改为有效的 DNS 服务器
2. 推荐使用：`8.8.8.8` 或 `114.114.114.114`
3. 如果是在 Cursor/Qoder 中配置，请修改相应的 DNS 设置
